Design pattern

  • 프로그램 개발에서 자주 나타나는 문제를 해결하기 위한 방법 중 하나로, 과거의 소프트웨어 개발 과정에서 발견된 설계의 노하우를 축적하여 이름을 붙여, 이후에 재이용하기 좋은 형태로 특정의 규약을 묶어서 정리한 것이다. wikipedia

Javascript Module pattern

  • class처럼 private, public를 캡슐화 하여 정의하는 방법이다
  • 별도 namespace 사용으로 global scope의 오염을 방지한다
  • Closure와 IIFE (Immediately Invoked Function Expression) 을 사용한다
  • 자바스크립트는 매우 유연한 언어이다. 이 언어를 모듈화 또는 OOP 하지 않는 것은 페라리를 출퇴근용으로만 사용하는 것과 같다.
  • 모듈화 하여 재사용 가능하도록 라이브러리를 구축하자
var Car = (fucntion(){
    var name    = '또봇';     //private variable
    var color   = 'blue';
    var state   = 'NONE';
    return{     //closure
        setName : function(newName){
            this.name = newName;
        },
        run : function(){
            this.state = "RUN";
            console.log(state);
        }
    };
})();   //IIFE
function Func1(x) {
    return function(y) {
        return x + y;
    };
}
function Func2() {
    var count=1;
    return function (cnt) {
        count += cnt;
    }
}

var f1 = Func1(5);
var f2 = Func1(10);
var f3 = Func2();

print(f1(2));   // ?
print(f2(2));   // ?
print(f3(10));  // ?
print(f3(10));  // ?
